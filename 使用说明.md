# 工业设备台账管理系统 - 安装与使用说明

---

## 一、系统简介

本系统是一套基于 **Python Flask + openGauss 数据库** 的工业设备全生命周期管理平台，采用 B/S 架构（浏览器/服务器模式），用户无需安装任何客户端软件，通过浏览器即可完成设备台账管理、维修记录追踪、车间数据统计以及 AI 故障预测。

**系统访问地址（已部署后！）：http://192.168.31.160:5000**

---

## 二、快速开始

### 2.1 访问系统

1. 确保电脑能访问 `192.168.31.160` 这个 IP 地址（与服务器在同一网络内）
2. 打开任意浏览器（Chrome、Edge、Firefox 均可）
3. 地址栏输入：`http://192.168.31.160:5000`
4. 页面加载后即可看到系统主界面

### 2.2 三个功能标签页

系统顶部有三个标签页，点击切换：

| 标签页 | 功能 |
|--------|------|
| 设备管理 | 查看/搜索/新增/编辑/删除设备，支持批量操作 |
| 维修记录 | 查看/筛选/新增维修记录 |
| 车间统计 | 查看车间统计表 + 设备维修趋势图 + AI 预测 |

## 三、服务器部署（管理员操作）

如果需要在新的服务器上重新部署，按以下步骤操作。

### 3.1 环境要求

| 组件 | 最低版本 | 说明 |
|------|----------|------|
| 操作系统 | CentOS 7 / Ubuntu 18.04+ | Linux 服务器 |
| Python | 3.10+ | 运行 Flask 应用 |
| openGauss | 5.0+ | 数据库，需含三张表 |
| pip3 | 22.0+ | Python 包管理器 |
| Git | 2.0+ | 用于克隆代码 |

### 3.2 数据库准备

确保 openGauss 数据库中存在以下三张表，并且 `ryan` 用户具有增删改查权限：

```sql
-- 1. 车间表
CREATE TABLE workshop (
    workshop_id   VARCHAR(20) PRIMARY KEY,
    workshop_name VARCHAR(50),
    location      VARCHAR(50)
);

-- 2. 设备表
CREATE TABLE equipment (
    equip_id           VARCHAR(20) PRIMARY KEY,
    equip_name         VARCHAR(100),
    model              VARCHAR(50),
    manufacture_date   TIMESTAMP,
    purchase_price     NUMERIC(12,2),
    workshop_id        VARCHAR(20),
    responsible_person VARCHAR(50),
    status             VARCHAR(20)
);

-- 3. 维修记录表（列存储，无外键约束）
CREATE TABLE maintenance (
    maintenance_id   SERIAL PRIMARY KEY,
    equip_id         VARCHAR(20),
    maintenance_time TIMESTAMP,
    fault_desc       TEXT,
    repair_cost      NUMERIC(10,2)
) WITH (orientation = column);

-- 4. 赋予权限
GRANT ALL PRIVILEGES ON ALL TABLES IN SCHEMA public TO ryan;
GRANT USAGE ON ALL SEQUENCES IN SCHEMA public TO ryan;
```

### 3.3 一键部署

SSH 登录到安装了 openGauss 的服务器，以 `omm` 用户身份执行：

```bash
# 1. 下载部署脚本
cd /home/omm
wget https://raw.githubusercontent.com/ydn6/equip-mgmt-system/master/deploy.sh

# 2. 执行部署
bash deploy.sh
```

脚本会自动完成以下所有操作：
- 从 GitHub 克隆项目代码
- 安装 Python 依赖包（flask、psycopg2-binary、faker）
- 修改数据库连接为服务器本地地址（127.0.0.1）
- 运行数据初始化脚本（插入 100 台设备 + 数千条维修记录）
- 在后台启动 Flask 应用（监听 0.0.0.0:5000）
- 放行防火墙 5000 端口

### 3.4 手动部署（如果一键脚本失败）

```bash
# 1. 克隆项目
cd /home/omm
git clone https://github.com/ydn6/equip-mgmt-system.git
cd equip-mgmt-system

# 2. 安装依赖
pip3 install flask psycopg2-binary faker

# 3. 修改数据库连接（编辑 app.py 第 16 行，将 192.168.31.160 改为 127.0.0.1）
sed -i 's/host="192.168.31.160"/host="127.0.0.1"/' app.py

# 4. 初始化数据
python3 rebuild_data.py

# 5. 后台启动
nohup python3 app.py > flask.log 2>&1 &

# 6. 验证
curl http://127.0.0.1:5000/api/workshops
```

### 3.5 日常管理命令

```bash
# 查看 Flask 是否在运行
pgrep -f "python3 app.py"

# 停止 Flask
pkill -f "python3 app.py"

# 查看日志
tail -f /home/omm/equip-mgmt-system/flask.log

# 重新启动
cd /home/omm/equip-mgmt-system
nohup python3 app.py > flask.log 2>&1 &
```

## 四、本地开发部署

如果需要在个人电脑上运行（用于二次开发或测试）：

### 4.1 克隆项目

```bash
git clone https://github.com/ydn6/equip-mgmt-system.git
cd equip-mgmt-system
```

### 4.2 创建虚拟环境（可选但推荐）

```bash
python -m venv venv

# Windows 激活
venv\Scripts\activate

# Linux/Mac 激活
source venv/bin/activate
```

### 4.3 安装依赖

```bash
pip install -r requirements.txt
```

### 4.4 配置数据库连接

用任意文本编辑器打开 `app.py`，找到第 10-18 行的 `get_db()` 函数，修改连接参数：

```python
def get_db():
    return psycopg2.connect(
        database="postgres",        # 数据库名
        user="你的用户名",            # 用户名
        password="你的密码",          # 密码
        host="你的数据库IP",           # 数据库服务器 IP
        port=5432                   # 端口
    )
```

### 4.5 初始化数据并启动

```bash
python rebuild_data.py   # 初始化 100 台设备 + 维修记录
python app.py            # 启动应用
```

看到 `Running on http://0.0.0.0:5000` 即启动成功，浏览器打开 `http://127.0.0.1:5000`。

## 五、详细使用指南

### 5.1 设备管理

#### 5.1.1 搜索设备

在顶部搜索框中输入设备编号（如 `001`）、设备名称（如 `数控车床`）或型号（如 `MOD-254`），点击"查询"按钮或按回车键，下方表格即显示匹配结果。

#### 5.1.2 新增设备

1. 点击右上角绿色"新增设备"按钮
2. 在弹出的表单中填写：
   - **设备编号**：3 位数字，如 `101`
   - **设备名称**：如 `数控车床`
   - **型号**：如 `MOD-500`
   - **出厂日期**：点击日历选择
   - **采购价格**：数字，单位元
   - **所属车间**：下拉选择已有车间
   - **负责人**：任意姓名
   - **状态**：未使用 / 在用 / 维修中 / 报废
3. 点击"保存"

#### 5.1.3 编辑设备

点击某设备行右侧的黄色"编辑"按钮，弹出窗口可修改负责人和状态。点击"保存"生效。

#### 5.1.4 批量编辑

1. 勾选需要修改的设备行左侧复选框（支持"全选"和"反选"按钮）
2. 点击"批量编辑"按钮
3. 在弹出的窗口中设置新的负责人和/或状态（留空则不改）
4. 点击"保存"，所有勾选的设备一次性更新

#### 5.1.5 删除设备

点击红色"删除"按钮。**删除前系统会校验**：
- 设备状态必须为"未使用"
- 设备不能有任何维修记录

不满足条件时会弹出提示，拒绝删除。这是为了防止误删正常使用中的设备数据。

---

### 5.2 维修记录

#### 5.2.1 查看维修记录

切换到"维修记录"标签页：

- **按设备查看**：在"设备"下拉框中选择一台设备，自动加载该设备所有维修记录
- **按时间范围筛选**：设置"起始月份"和"截止月份"后自动筛选
- **按故障类型搜索**：在"故障描述"框中输入关键词（如"主轴异响"、"电路故障"），按回车搜索

以上筛选条件可以组合使用。例如：选择设备为 `050`、起始月份 `2025-01`、故障描述 `润滑不足`，即可精确查找该设备在 2025 年间所有润滑相关故障。

#### 5.2.2 新增维修记录

1. 点击右上角绿色"新增维修记录"按钮
2. 选择设备、填写维修时间、故障描述、维修费用
3. 点击"保存"

**注意**：维修费用不能为负数，系统会在保存前校验。

#### 5.2.3 清空筛选

点击筛选面板右上角"清除"按钮，恢复显示全部维修记录。

---

### 5.3 车间统计

#### 5.3.1 车间统计表

切换到"车间统计"标签页，上方表格显示 5 个车间的汇总数据：

| 列名 | 含义 |
|------|------|
| 车间名称 | 加工车间、装配车间等 |
| 设备总数 | 该车间所有设备数量 |
| 在用设备 | 状态为"在用"的设备数 |
| 维修总次数 | 该车间所有设备的历史维修次数合计 |

#### 5.3.2 设备维修趋势图

1. 在"设备维修趋势"输入框中输入设备编号（如 `050`）
2. 点击"查看趋势"按钮或按回车键
3. 页面显示该设备的月度维修次数柱状图

**图表特性**：
- 蓝色渐变柱状图：历史维修数据
- 橙红色虚线柱状图（最后一根）：DB4AI 预测的下月维修次数
- 鼠标悬停柱体可查看详细数值和置信区间
- 图表标题会显示预测方法（DB4AI 或回退算法）

#### 5.3.3 预测原理

系统优先调用 openGauss 内置的 **DB4AI.PREDICT_TIME_SERIES** 函数进行时序预测；如果 DB4AI 模块未安装，则自动回退到 **加权移动平均算法**（近期数据权重更高），同时提供置信区间供参考。

## 七、API 接口文档

所有接口返回 JSON 格式数据。

### 7.1 车间

| 方法 | URL | 说明 | 参数 |
|------|-----|------|------|
| GET | `/api/workshops` | 车间列表 | 无 |

### 7.2 设备

| 方法 | URL | 说明 | 参数 |
|------|-----|------|------|
| GET | `/api/equipment` | 搜索设备 | `?q=` 编号/名称/型号（可选，不传则返回全部）|
| POST | `/api/equipment` | 新增设备 | JSON body: equip_id, equip_name, model, manufacture_date, purchase_price, workshop_id, responsible_person, status |
| PUT | `/api/equipment/<id>` | 更新设备 | JSON body: responsible_person, status |
| DELETE | `/api/equipment/<id>` | 删除设备 | 仅"未使用"且无维修记录可删 |
| PUT | `/api/equipment/batch` | 批量更新 | JSON body: { ids: [...], updates: { responsible_person?, status? } } |

### 7.3 维修记录

| 方法 | URL | 说明 | 参数 |
|------|-----|------|------|
| GET | `/api/maintenance/<id>` | 设备维修记录 | URL 路径中的设备编号 |
| GET | `/api/maintenance/search` | 多条件筛选 | `?equip_id=&date_from=&date_to=&fault_desc=` 全部可选 |
| POST | `/api/maintenance` | 新增维修 | JSON body: equip_id, maintenance_time, fault_desc, repair_cost |

### 7.4 统计与预测

| 方法 | URL | 说明 | 参数 |
|------|-----|------|------|
| GET | `/api/stats/workshop` | 车间统计 | 无 |
| GET | `/api/stats/trend` | 月度趋势 | `?equip_id=` 设备编号 |
| GET | `/api/stats/predict` | DB4AI 预测 | `?equip_id=` 设备编号 |
